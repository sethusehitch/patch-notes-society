# Patch Notes for Society

First-principles breakdowns of public systems we inherited, the bugs they create, and the upgrades worth testing next.

This is a civic systems project, not a claim that first drafts solve hard public problems. The format is: history, incentives, failure modes, tradeoffs, metrics, migration plans, and revision based on critique.

## Review Wanted

Useful feedback is specific:

- What is wrong or incomplete?
- Which incentive is missing?
- Which reform would backfire?
- What metric would be gamed?
- What source, case study, or operator reality should be added?
- Which paper should become a formal policy memo first?

Use the public feedback forms for critique, source corrections, implementation risks, or general project feedback:

https://github.com/sethusehitch/patch-notes-society/issues/new/choose

## Status

These are public v0.1 drafts for critique and revision, not final policy submissions. The policy track is separate: evidence matrices, reviewer packets, and submission-grade memos under The Better Systems Project.
